Ветвления: if / elif / else

  • Условие — выражение, которое даёт True или False.
  • Отступы определяют блок кода (обычно 4 пробела).
  • elif — “иначе если” (можно много раз), else — “иначе”.
x = int(input())
if x > 0:
    print("плюс")
elif x == 0:
    print("ноль")
else:
    print("минус")

Полезные паттерны

  • Проверка диапазона: 0 <= x <= 10
  • Логика: and, or, not
  • Тернарный оператор: msg = "ok" if x >= 18 else "no"

Циклы: for / while

for + range

for i in range(5):
    print(i)  # 0 1 2 3 4
  • range(n) — 0..n-1
  • range(a, b) — a..b-1
  • range(a, b, step) — с шагом

while

x = 5
while x > 0:
    print(x)
    x -= 1

break / continue

for x in [1,2,3,4,5]:
    if x == 3:
        continue  # пропустить 3
    if x == 5:
        break     # остановиться
    print(x)
Самая частая ошибка новичка в while — забыть менять переменную → бесконечный цикл.

Функции

  • Функция — “упакованный кусок кода”, который можно вызвать много раз.
  • return возвращает значение и заканчивает функцию.
  • Аргументы бывают позиционные и именованные.
def is_even(x):
    return x % 2 == 0

print(is_even(10))  # True

Значения по умолчанию

def greet(name="гость"):
    return f"Привет, {name}!"

print(greet())
print(greet("Иван"))
Мини-правило: функция должна делать одну понятную вещь и иметь понятное имя.

Lambda-функции (очень кратко)

lambda — это “короткая функция в одну строку”. Часто используется как ключ сортировки или как “передать маленькое правило”. 
nums = [5, 2, 9]
nums_sorted = sorted(nums)               # [2,5,9]
pairs = [("a", 3), ("b", 1)]
pairs_sorted = sorted(pairs, key=lambda x: x[1])  # сортируем по второму

Ограничение

  • lambda — одно выражение, без многострочных if/for.

ООП

  • Класс — шаблон/описание “объекта”.
  • Объект — конкретный экземпляр класса.
  • self — ссылка на текущий объект.
class Counter:
    def __init__(self, start=0):
        self.value = start

    def inc(self):
        self.value += 1
        return self.value

c = Counter(10)
print(c.inc())  # 11
Для управленцев достаточно понимать: “в объекте лежат данные + есть методы, которые с ними работают”.

Исключения: try / except

  • Исключение — это “ошибка во время выполнения”.
  • Мы можем её перехватить и обработать, чтобы программа не падала.
try:
    x = int(input())
    print(10 / x)
except ValueError:
    print("Нужно ввести число!")
except ZeroDivisionError:
    print("На 0 делить нельзя!")

Когда использовать

  • Когда ввод пользователя может быть неправильным.
  • Когда читаем файлы/сеть (может не быть доступа).
В учебных задачах важно научиться читать текст ошибки и понимать: “что сломалось” → “почему” → “как исправить”.
